# Сетевая модель российской власти: формальные институты, кадровые связи и аналитические слои влияния

## Идея проекта

Проект исследует, как соотносятся **формальная архитектура российской власти** и **сетевые структуры кадровых, координационных и аналитически реконструируемых связей**.

Ключевой принцип проекта: финальный результат сделан **собственным Python-кодом**, а не в сервисе визуализации. Таблицы Kumu используются только как один из исходных форматов хранения данных, после чего проходят кодовую предобработку, объединение, анализ и визуализацию.

## Исследовательский вопрос

**Насколько формальная структура российской власти совпадает с сетевой структурой кадровых, координационных и аналитически реконструируемых связей?**

## Исследовательские задачи

1. Собрать и нормализовать формальный институциональный слой российской власти.
2. Сопоставить формальный institutional core с аналитическими слоями `overlay` и `comention`.
3. Выявить наиболее центральные узлы в формальном ядре сети.
4. Показать, какие типы связей доминируют в разных слоях данных.
5. Сравнить формальную и аналитически реконструированную архитектуру сети.
6. Обсудить ограничения verified-only данных и исследовательской реконструкции.

## Данные

### 1. Формальный институциональный слой
Проект включает **автоматизированный сбор с официальных сайтов**:
- Кремль (`kremlin.ru`)
- Правительство РФ (`government.ru`)
- Государственная Дума (`duma.gov.ru`)
- Совет Федерации (`council.gov.ru`)

Модуль `src/scrape_official.py` пытается собрать live-данные с этих страниц. Для офлайн-воспроизводимости в репозиторий включен **кэшированный snapshot официального слоя** (`data/raw/official_collection/official_snapshot_cache.csv`).

Это позволяет честно показать, что институциональная часть проекта не сводится к ручной Kumu-разметке: формальный слой может быть обновлен кодом с официальных источников.

### 2. Исследовательский корпус
Используются исходные CSV-таблицы, первоначально подготовленные для Kumu:
- `kumu_core_elements_import_type.csv`
- `kumu_core_connections_import_type.csv`
- `kumu_overlay_elements_import_type.csv`
- `kumu_overlay_connections_import_type.csv`
- `kumu_comention_elements_import_type.csv`
- `kumu_comention_connections_import_type (1).csv`

В репозитории они используются не как финальная визуализация, а как **сырой корпус данных**, который обрабатывается Python-кодом.

### 3. Слои данных

Изначально часть данных хранилась в Kumu-совместимом формате, однако итоговый проект полностью реализован в Python. Kumu использовался только как промежуточный формат организации данных, а анализ, объединение слоев, расчет метрик и визуализация выполнены собственным кодом

- **official** — формальный институциональный слой, собранный/обновляемый с официальных сайтов;
- **core** — формальные узлы и связи исследовательской базы;
- **overlay** — аналитический слой, собранный по исследованиям о сетевых структурах и неформальных элитных связях;
- **comention** — дополнительный слой соупоминаний из внешнего корпуса сообщений.

## Предобработка данных

1. Автоматизированный сбор или загрузка кэшированного официального snapshot.
2. Импорт всех CSV.
3. Добавление признака слоя (`layer`).
4. Объединение таблиц узлов и таблиц связей.
5. Нормализация переменных.
6. Построение графов:
   - formal core
   - overlay
   - comention
   - merged graph
7. Подсчет сетевых метрик для formal core.

## Методы

### 1. Автоматизированный сбор данных
Используется модуль `src/scrape_official.py`, который:
- собирает формальный институциональный слой с официальных сайтов;
- сохраняет воспроизводимый snapshot;
- позволяет отделить verified-only институциональные данные от аналитической надстройки.

### 2. Сетевой анализ
Используются:
- degree centrality
- betweenness centrality
- closeness centrality
- распределение связей по типам
- визуализация подграфа наиболее связанных узлов

### 3. Сравнительный анализ многослойной сети
Слои `official`, `core`, `overlay` и `comention` сопоставляются по:
- числу узлов и связей;
- типам связей;
- отличию формального и аналитического каркаса;
- роли аналитически добавленных узлов.

## Ограничения

1. Формальный институциональный слой собирается с официальных сайтов, но полный исследовательский граф не является purely official.
2. Слой `overlay` не эквивалентен strictly verified-only данным.
3. Слой `comention` отражает не власть как таковую, а совместную видимость/связность в корпусе сообщений.
4. Часть данных изначально была подготовлена в Kumu-формате, но затем перенесена в Python-пайплайн.
5. Live-сбор с официальных сайтов зависит от доступности сети; поэтому в репозиторий включен кэшированный snapshot для офлайн-проверки.
